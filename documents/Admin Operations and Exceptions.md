## Purpose

This document outlines the **Admin Operations and Exceptions** workflow.

While the Principal manages Academic and Reputational reality, the Admin manages the sheer, grinding logistical and financial reality of the institution. They work primarily alongside the `Admin Agent` to maintain `Layer 1 & 2 Setup Architecture`.

---

## 1. The Backstage Setup Drift (The Routine Workflow)

The Admin is responsible for keeping the physical infrastructure (venues, rosters, fees) perfectly aligned with reality, so the Autonomy Engines don't hallucinate.

### The Substrate Cleansing Loop
*   **Draft**: A new student enrolls mid-semester.
*   **Action**: The Admin opens the Setup Workspace and injects the student into a Section.
*   **AI Dependency Spawning**: The `Admin Agent` instantly detects the new entity. It automatically:
    1. Attaches the standard fee structure to the student.
    2. Modifies the Class 10 roster.
    3. Triggers the parents' onboarding `Awareness Card`.

### Fee Operations
Mintrix attempts to run all fee collection continuously in `Operator` mode (sending auto-reminders based on the ledger). 
*   **The Admin Intervention**: When a parent disputes a fee or negotiates a manual concession, the Admin overrides the `Operator` tracking via the Financial Workspace, manually inputting a receipt that recalibrates the ledger.

---

## 2. The Spawning Event Workflow

When a major physical disruption occurs (like a massive off-site examination event or a holiday), the Admin uses the `Event Architecture` system.

1.  **Creation**: Admin opens the `Event Spawner Workspace`.
2.  **Definition**: They define the type as "Standardization Exam."
3.  **Conflict Checking**: The AI flashes the `Intelligence Sidebar` red—*The desired venue overlaps with 4 physical education classes.*
4.  **Triage**: Admin forcefully creates new temporary venue assignments.
5.  **Execution**: The Admin Agent fires 40 parallel workflows (Notifying PE teachers, shifting timetables, blocking the auditorium calendar). 

---

## 3. Administrative Exceptions

The Admin `Exception Center` is primarily triggered by **External Validation Failures**.

*   **Example Exception**: A parent payment bounces via the API integration. 
*   **AI Reaction**: The system cannot mathematically reconcile a bounced API check against an issued internal invoice. It flags an Amber Exception. The Admin must manually untangle the ledger and re-issue the penalty.
