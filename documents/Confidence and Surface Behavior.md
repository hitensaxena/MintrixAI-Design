## Purpose

This document defines **Confidence and Surface Behavior**. 

In Mintrix, the UI is not static. It visually warps, color-shifts, and flags users based on a singular internal mathematical metric: **The Confidence Score (0-100%)**. 

This document outlines how that invisible score is translated into human-readable UI patterns across all architectural surfaces.

---

## 1. The Confidence Metric

Every piece of data generated by the `Intelligence Layer Processor` carries a tag from 0 to 100. 

*   **100% (Absolute Certainty):** System possesses multi-layered historical verification and explicit Setup Policy alignment.
*   **70-99% (High Confidence):** System has strong statistical probability but acknowledges minor operational drift.
*   **40-69% (Moderate/Warning):** System detects a structural conflict (e.g., A venue is double booked) or lacks historical context.
*   **0-39% (Low/Risk):** System is hallucinating due to missing `Setup Architecture` data or severe structural failure.

---

## 2. Confidence Badges & Visual Language

When a `Persona Agent` outputs a recommendation or drafts a workflow, the UI must explicitly broadcast the Confidence Score using a standardized badge system. We do not hide the AI's uncertainty; we highlight it as a feature of Trust.

<FeatureGrid>

<SurfaceCard title="The Verified Badge (90-100%)">
**Visuals**: Solid Green dot or "Sparkle" icon. Minimal border. 
**Behavior**: The UI element feels calm. If this is an `Operator` workflow, it executes silently in the background, only leaving a receipt in the Transparency Log.
**Text Prompt**: *System Verified* or *Routine Execution*.
</SurfaceCard>

<SurfaceCard title="The Judgment Badge (50-89%)">
**Visuals**: Amber/Yellow dot. Active "Hover to View Why" affordance.
**Behavior**: The UI halts the user. It explicitly forces the workflow into `Collaborator` mode, dropping a `Judgment Card` into the Approval Inbox. It requires an explicit human signature.
**Text Prompt**: *Review Recommended* or *Historical Deviation Detected*.
</SurfaceCard>

<SurfaceCard title="The Calibration Warning (< 50%)">
**Visuals**: Pulsing Red border. Aggressive icon (Triangle/Exclamation).
**Behavior**: The UI refuses to act. It drops the workflow into the `Exception Center`, effectively downgrading the AI to a `Tool`. It demands manual human overrides.
**Text Prompt**: *Calibration Required* or *Policy Conflict*.
</SurfaceCard>

</FeatureGrid>

## 3. Surface-Specific Warping

The Confidence Score dictates exactly how the 6 Core Surface Families adapt visually.

### In the Daily Feed (Ambient)
*   **High Confidence**: Cards are compressed, taking up minimal Y-axis height. E.g., *"3 absent notices sent."*
*   **Low Confidence**: The Feed refuses to show low-confidence data to avoid polluting ambient awareness. It routes that data directly to the Inbox.

### In the Workspace (Execution)
The `Intelligence Sidebar` is highly reactive to the Confidence Score as the user types.
*   **Real-time Shift**: If a Teacher types *"Schedule a test for tomorrow,"* and tomorrow is a School Holiday (Layer 2 Setup failure), the sidebar flashes **Amber**, disabling the "Submit" button until the Teacher checks a "Force Override" box.

### In the Approval Inbox (Action)
*   **Comparison View Priority**: Judgment Cards in the Inbox are sorted *by* lowest confidence first, not just by timestamp. The system forces humans to triage the most uncertain AI drafts immediately to prevent operational gridlock.

---

## 4. Reversibility & The "Undo" State

Surface behavior must also reflect the permanence of an action.

*   If an `Operator` executes a High-Confidence action (e.g., sending a mass email), the `Transparency Log` automatically applies an "Active State" to the record, keeping a bright **Undo** button active for the duration of the configured Reversibility Window (e.g., *15 minutes*).
*   Once the window closes, the UI surface physically fades the Undo button to gray, transitioning the record from an "Active State" to an "Immutable Historical State."
