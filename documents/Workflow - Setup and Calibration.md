## Purpose

This document defines the **Setup and Calibration Workflow** in Mintrix. This is the foundation upon which all intelligence is built.

Mintrix does not treat setup as a one-time "Onboarding" form; it treats it as a **progressive trust-building process** that continues long after the first login.

---

## 1. The Setup Architecture: The Five-Layer Model

The system is configured in five distinct layers, each adding a new dimension to its intelligence.

| Layer | Component | Significance |
| :--- | :--- | :--- |
| **Layer 1** | **Institutional Structure** | Campus, Grades, Sections, Hierarchy. (The Skeleton) |
| **Layer 2** | **Academic Model** | Subjects, Syllabus, Timetable Backbone. (The Brain) |
| **Layer 3** | **People & Permissions** | Roles, Authority, Parent/Student Data. (The Actors) |
| **Layer 4** | **Operational Logic** | Attendance Rules, Fee Models, Notice Routing. (The Habits) |
| **Layer 5** | **Live Operational Hooks** | Event Templates, Substitutions, Ad-hoc Payments. (The Response) |

---

## 2. The Setup Lifecycle

1.  **Preparation (Foundation)**: Admin completes Layers 1 and 2.
2.  **Minimum Viable Intelligence (Launch)**: System provides basic briefings and alerts (Day 1).
3.  **Calibration Phase (Learning)**: System observes the "School's Rhythm" for 2-4 weeks.
4.  **Operator Maturity (Scale)**: System moves workflows from "Assistant" to "Operator" as confidence grows.

---

## 3. The Calibration Panel (Confidence Surface)

The **Calibration Surface** is where the system tells the school what it's learning and where its data is "thin."

-   **Data Gap Visibility**: "I am 40% confident in Grade 5 syllabus tracking because I have missing timetable data."
-   **Confirmation Loops**: "I've noticed Class 8A often starts 10 minutes late. Is this a permanent schedule shift?"
-   **Success Signals**: "Calibration complete for Class 5A. Attendance alerts are now moving to Operator mode."

---

## 4. Interaction Patterns for Setup

| Pattern | Description |
| :--- | :--- |
| **The "Confidence Gauge"** | A visual indicator (Red/Amber/Green) showing how ready a workflow is for automation. |
| **The "Missing Context" Prompt** | Proactive nudges in the Daily Feed for missing data (e.g., *"Help me understand this substitution rule"*). |
| **The "Setup Progress" Board** | A visual map of the five layers and their completion status. |

---

## 5. Autonomy Shifts During Calibration

| Phase | Autonomy Mode | System Behavior |
| :--- | :--- | :--- |
| **Initial Setup** | **Tool** | System only acts when explicitly commanded. |
| **Early Calibration** | **Assistant** | System prepares context and asks for confirmation often. |
| **Late Calibration** | **Collaborator** | System suggests actions with high confidence. |
| **Post-Calibration** | **Operator** | System handles routine actions autonomously and logs them. |

---

## 6. Success Criteria: "Believeable from Day 1"

The goal of this workflow is to avoid the "Blank State" problem. Even with minimal setup (Layer 1), the system should feel useful. As the Admin closes "Data Gaps," the system rewards them with increased autonomy and deeper insights.
